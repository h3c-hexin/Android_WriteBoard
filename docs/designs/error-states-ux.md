## 1. 错误提示组件总览

PaintBoard 使用三种错误提示形式，按干扰程度从低到高：

| 组件 | 形式 | 使用场景 | 是否阻塞操作 |
|------|------|---------|-----------|
| **Toast** | 屏幕底部短暂弹出，自动消失 | 轻量信息通知（成功/失败/提示），2~3s 后自动消失 | 否 |
| **Banner** | 画布顶部固定横条，持续显示 | 持续状态变化（协同断线、重连中），需要用户持续感知 | 否 |
| **Bottom Sheet（Error）** | 从底部弹出，需用户主动关闭 | 需要用户决策的错误（文件保存失败、格式错误），附操作按钮 | 是（需操作后关闭） |

### 1.1 Toast 规范

```
位置：屏幕底部，距工具栏上方 16dp
高度：48dp（大屏）/ 40dp（PAD）
水平内边距：24dp
圆角：radius.full（胶囊形）
背景：rgba(62, 62, 62, 0.92)（深色半透明，深色主题）
文字：typography.size.body（18sp 大屏），color.icon.on-surface（#EEEEEE）
图标（可选）：左侧 20dp 图标
持续时长：普通 Toast 2s，重要 Toast 3s
动画：底部淡入（translateY: +20dp → 0，opacity: 0 → 1，200ms）
      自动消失（opacity: 1 → 0，150ms）
```

### 1.2 Banner 规范

```
位置：画布顶部（紧贴屏幕顶边），宽度全屏
高度：28dp（大屏）/ 24dp（PAD）
内边距：水平 16dp，垂直 4dp
文字：typography.size.caption（14sp 大屏），单行，水平居中
图标：左侧 14dp 状态图标，与文字间距 6dp
透明度：80%（视觉轻量，不完全遮挡画布内容）

Banner 不含任何交互按钮（仅状态展示），
例外：断线超 5 次失败后 Banner 变为可点击（见 2.3 节）
```

### 1.3 Error Bottom Sheet 规范

```
样式同通用 Bottom Sheet，但：
  标题图标：⚠️ 或 ❌（根据严重程度）
  标题颜色：color.semantic.error（#EF9A9A，深色模式）
  内容文字：color.icon.on-surface（#EEEEEE）
  操作按钮区：底部横排，主要操作在右
  关闭方式：点击按钮（点击遮罩不关闭，要求用户明确选择操作）
```

## 2. 协同断线提示

### 2.1 触发条件

| 触发条件 | 端 |
|---------|---|
| PAD WebSocket `onClose` / `onError` 回调触发 | PAD 端 |
| Host 检测某 PAD 心跳超时（5s 内无 PONG） | Host 端 |
| Host WebSocket 服务意外停止 | 所有端 |

### 2.2 PAD 端断线提示流程

```
WebSocket 连接断开（PAD 端）
  │
  ▼
  即时触发（0ms 延迟）：
    工具栏协同按钮徽章变为橙色 + 脉冲动画
    画布顶部 Banner 切换为橙色（覆盖原绿色 Banner）
  │
  ▼
  Banner 内容（断线第 1~4 次重连）：
    背景色：color.semantic.warning.container（#E65100，深色模式）
    图标：🟠
    文字："协同已断线 · 正在重新连接... 第 N 次"
    脉冲动画：Banner 背景透明度在 60%~100% 循环（周期 1s）
  │
  ▼
  （重连成功 → 见 collaboration-ux.md 6.3 节）
  │
  ▼
  （重连 5 次仍失败后）：
    Banner 内容更新：
    背景色：color.semantic.error.container（#B71C1C，深色模式）
    图标：🔴
    文字："协同连接失败 · 点击手动重连"
    Banner 变为可点击区域（整个 Banner 高度内均可触摸）
    点击触发立即重连
```

### 2.3 Banner 状态一览

| 协同状态 | Banner 背景色 | 图标 | 文字 | 可点击 |
|---------|------------|------|------|--------|
| 协同已开启，连接正常 | `color.semantic.success.container`（#1B5E20） | 🟢 | "协同中 · N 台设备在线" | 否 |
| 断线，重连中（< 5 次） | `color.semantic.warning.container`（#E65100） | 🟠 | "协同已断线 · 正在重新连接... 第 N 次" | 否 |
| 断线，重连失败（≥ 5 次） | `color.semantic.error.container`（#B71C1C） | 🔴 | "协同连接失败 · 点击手动重连" | 是 |
| 断线，已放弃重连 | Banner 消失 | — | — | — |

### 2.4 Host 端通知 PAD 断线

当 Host 检测到某台 PAD 断线时：
- **不弹出任何弹窗**（避免打断 Host 的书写）
- 协同 Bottom Sheet 的在线设备列表中，该设备条目显示断线状态（橙色小点）
- 工具栏协同按钮徽章在线人数数字减少（减去断线设备数）
- Toast（轻量通知）："PAD-2 已断线"，持续 3s

### 2.5 协同会话被 Host 结束时（PAD 端）

- Toast："大屏已结束协同"，持续 3s
- 画布顶部绿色 Banner 淡出消失（500ms）
- 协同按钮徽章清除
- PAD 本地内容保留（不清空）

## 3. 文件保存失败提示

### 3.1 触发条件

| 触发条件 | 说明 |
|---------|------|
| SAF 写入 URI 失败 | 存储空间不足、权限被撤销、目标路径不可写 |
| .pb 序列化失败 | 数据结构异常（极少发生） |
| 用户取消系统文件选择器 | 用户主动取消，非错误但需清理中间状态 |

### 3.2 保存失败提示 UI

**用户取消文件选择器**（非错误）：
- 静默处理，关闭命名 Bottom Sheet，回到白板
- 不弹任何提示

**写入失败（存储空间不足）**：
- 进入"存储空间不足"专项错误流程（见第 4 节）

**写入失败（权限 / 其他 I/O 错误）**：

```
保存失败时，在命名 Bottom Sheet 底部显示 Error Banner：

  ┌──────────────────────────────────────────────────────────────┐
  │  ╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌（拖拽条）                             │
  │                                                              │
  │  保存白板                                                     │
  │  ┌──────────────────────────────────────────────────────┐    │
  │  │ ❌  保存失败：无法写入文件，请检查存储权限              │    │  ← Error Banner
  │  └──────────────────────────────────────────────────────┘    │
  │                                                              │
  │  文件名                                                       │
  │  ┌──────────────────────────────────────────────────────┐    │
  │  │  会议_2026-03-23_14:32                              ✕ │    │
  │  └──────────────────────────────────────────────────────┘    │
  │                                                              │
  │               [ 取消 ]    [ 重试 → 重新选择位置 ]               │
  │                                                              │
  └──────────────────────────────────────────────────────────────┘
```

Error Banner 规范：
- 背景：`color.semantic.error.container`（`#B71C1C`，深色模式）
- 图标：❌，尺寸 `20dp`，颜色 `color.semantic.error`
- 文字：`typography.size.body`（18sp），`color.semantic.error`
- 圆角：`radius.sm`（8dp）
- 内边距：`12dp` 上下，`16dp` 左右

### 3.3 恢复操作

| 恢复操作 | 说明 |
|---------|------|
| 点击【重试 → 重新选择位置】 | 重新调用 SAF 文件选择器，用户可选择其他存储位置 |
| 点击【取消】 | 关闭 Bottom Sheet，内容未保存，但保留画布内容 |
| isDirty 状态 | 保存失败后 isDirty 保持 `true`，保证用户不会误以为已保存 |

## 4. 存储空间不足提示

### 4.1 触发条件

| 触发场景 | 检测时机 |
|---------|---------|
| 保存 .pb 文件 | 写入 URI 前预检，或写入时捕获 `IOException: No space left` |
| 导出 PNG / PDF | 生成临时文件时捕获磁盘空间异常 |
| 崩溃恢复自动保存 | 写入 `recovery_tmp.pb` 时失败（静默，不触发 UI 提示） |

**预检逻辑**（写入前主动检查）：
```kotlin
val estimatedSize = PageDataList.estimatedBytes()
val freeSpace = File(targetDir).freeSpace
if (freeSpace < estimatedSize * 1.5) {
    // 触发存储空间不足提示
}
```

### 4.2 存储空间不足 Bottom Sheet

```
┌──────────────────────────────────────────────────────────────┐
│  ╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌（拖拽条）                             │
│                                                              │
│  ⚠️  存储空间不足                                             │
│                                                              │
│  设备可用空间不足，无法保存文件。                                │
│  当前可用：XX MB（不足 YY MB）                                 │
│                                                              │
│  建议：                                                       │
│  • 删除不需要的文件以释放空间                                   │
│  • 通过扫码分享（📤分享按钮）将白板发送给手机                    │
│                                                              │
│              [ 关闭 ]       [ 去设置释放空间 ]                  │
│                              （跳转到 Android 存储管理页）      │
└──────────────────────────────────────────────────────────────┘
```

### 4.3 各触发场景的提示文案

| 触发场景 | 标题 | 说明文字 |
|---------|------|---------|
| 保存 .pb 文件 | "存储空间不足，无法保存" | "白板无法保存，建议通过"扫码分享"先将内容发给手机保存。" |
| 导出 PNG | "存储空间不足，无法导出" | "图片导出失败，可用空间不足 {X} MB。" |
| 导出 PDF | "存储空间不足，无法导出" | "PDF 生成失败，可用空间不足 {X} MB。" |

### 4.4 恢复操作

| 按钮 | 行为 |
|------|------|
| 关闭 | 关闭 Bottom Sheet，内容保留在画布 |
| 去设置释放空间 | 调用 `Settings.ACTION_INTERNAL_STORAGE_SETTINGS` 跳转到 Android 存储管理 |

## 5. 扫码分享失败提示

### 5.1 HTTP 服务启动失败

**触发条件**：端口 8080~8082 全部被占用，无法绑定端口。

```
提示方式：分享 Bottom Sheet 内 Error Banner
          （不使用全屏弹窗，不阻断用户操作）

┌──────────────────────────────────────────────────────────────┐
│  ❌  分享服务启动失败，请稍后重试                                 │
│  端口被占用，可尝试关闭其他应用后重试。                           │
│                                        [ 重试 ]              │
└──────────────────────────────────────────────────────────────┘
```

### 5.2 无 WiFi 连接

**触发条件**：获取局域网 IP 失败（设备未连接 WiFi，仅有移动数据）。

```
提示方式：Toast（非阻塞，2.5s）

文案：
  "未连接 WiFi，扫码分享需要局域网。请先连接 WiFi。"
  图标：📶（无信号）

无分享 Bottom Sheet 弹出（直接拦截在按钮点击阶段）
```

### 5.3 页面截图 / PDF 生成失败

**触发条件**：
- 内存不足，画布截图 OOM（OutOfMemoryError）
- PDF 生成时系统 PDFDocument 异常

```
提示方式：分享 Bottom Sheet 内 Error Banner

文案（PNG 截图失败）：
  "页面截图失败，内存不足。请关闭其他应用后重试。"
  操作按钮：[ 重试 ]

文案（PDF 生成失败）：
  "PDF 生成失败，请稍后重试或导出为 PNG 格式。"
  操作按钮：[ 切换为 PNG ]   [ 重试 ]
```

### 5.4 Token 失效（手机端访问时）

此场景在手机浏览器端处理（HTTP 服务响应），大屏 App 端无感知，无需 App 端 UI 提示。

浏览器端显示（HTTP 服务返回 403 页面）：
```html
<h2>此分享链接已失效</h2>
<p>请在大屏上重新点击"分享"按钮，生成新二维码。</p>
```

## 6. 其他错误场景

### 6.1 打开文件格式错误

**触发条件**：用户选择了非 `.pb` 格式文件，或 `.pb` 文件版本过旧 / 数据损坏。

| 情况 | 提示方式 | 文案 |
|------|---------|------|
| 文件格式不支持 | Toast（3s） | "无法打开此文件，不是有效的 PaintBoard 白板文件（.pb）" |
| 文件版本过新（App 需升级） | Bottom Sheet | "此文件由更新版本的 PaintBoard 创建，请升级 App 后打开。" |
| 文件损坏 | Bottom Sheet | "文件读取失败，文件可能已损坏。" |

**文件损坏 Bottom Sheet**：
```
┌──────────────────────────────────────────────────────────────┐
│  ╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌（拖拽条）                             │
│                                                              │
│  ❌  文件无法打开                                              │
│                                                              │
│  文件读取失败，文件可能已损坏或格式不兼容。                       │
│  当前白板内容保持不变。                                         │
│                                                              │
│                            [ 确认 ]                          │
└──────────────────────────────────────────────────────────────┘
```

### 6.2 撤销 / 重做到达边界

**不弹提示**，仅通过按钮禁用态（`color.icon.disabled`）视觉传达已达到边界。（轻量处理，不打扰用户书写）

### 6.3 新增页达到上限

| 触发位置 | 提示方式 | 文案 |
|---------|---------|------|
| 工具栏 ＋ 按钮 | Toast（2s） | "已达最大页数（20 页）" |
| 页面管理 Bottom Sheet "新增页"卡片 | Toast（2s，不关闭 Bottom Sheet） | "已达最大页数（20 页）" |

### 6.4 协同加入失败

| 失败原因 | 提示方式 | 文案 | 恢复操作 |
|---------|---------|------|---------|
| 连接超时（3s 内无响应） | Bottom Sheet Error Banner | "连接超时，请确认与大屏连接到同一 WiFi。" | [ 重试 ] |
| Token 验证失败 | Bottom Sheet Error Banner | "加入失败，二维码已失效，请让大屏重新显示二维码后扫码。" | [ 重新扫码 ] |
| 网络不可用 | Toast（3s） | "无网络连接，请检查 WiFi 设置。" | — |

### 6.5 图形自动矫正超时

**触发条件**：ML Kit 识别超过 500ms 未返回结果。

- **静默处理**：不弹任何提示，直接保留原始手绘图形
- 用户不感知（效果上等同于"不矫正"，符合预期）
- 不弹 Toast，避免频繁提示干扰书写节奏

## 7. 错误提示设计原则

### 7.1 错误提示的四要素

每条错误提示均包含：

| 要素 | 说明 |
|------|------|
| **触发条件** | 明确何时显示（开发实现依据） |
| **提示文案** | 简洁说明出了什么问题（用户理解） |
| **恢复操作** | 提供明确的下一步（用户行动） |
| **严重程度** | 决定提示形式（Toast / Banner / Bottom Sheet） |

### 7.2 严重程度与提示形式选择

```
严重程度判断树：

  用户是否需要立即做出决策？
    ├─ 否 → 是否需要持续感知？
    │         ├─ 否 → Toast（轻量通知）
    │         └─ 是 → Banner（持续状态）
    └─ 是 → Bottom Sheet（需要用户操作）
```

### 7.3 文案写作原则

- **避免技术术语**：不使用 "IOException"、"HTTP 500"、"NullPointerException" 等
- **说明原因**：优先说明"为什么"而不是"发生了什么"（如"未连接 WiFi"而非"网络错误"）
- **给出出路**：每条错误提示必须有恢复操作或合理的关闭方式
- **保持简短**：Toast 最多 1 行，Bottom Sheet 说明最多 3 行

### 7.4 不打断书写原则

- 书写过程中（画布激活触摸中）**禁止弹出阻塞性 Bottom Sheet**
- 协同断线等异步错误仅通过 Banner 或按钮状态指示，不打断当前笔画
- 自动保存失败、矫正超时等后台错误**完全静默**（不通知用户）

### 7.5 错误提示层级

```
同时存在多个错误时，按以下优先级处理：

  P1  崩溃恢复（启动时一次性，阻塞操作）
  P2  操作级错误（需用户决策，Bottom Sheet）
  P3  持续状态错误（协同断线，Banner）
  P4  轻量通知（存储操作结果，Toast）

同一时刻：
  最多显示 1 个 Bottom Sheet
  最多显示 1 个 Banner
  Toast 可与 Banner 并存（Toast 在 Banner 下方）
  Toast 排队显示，不叠加
```
