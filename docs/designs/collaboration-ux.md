## 1. 协同架构概述

```
局域网协同架构（同一 WiFi 下）：

  大屏（Host）                          PAD（Participant）
  ─────────────────                    ─────────────────────
  运行本地 WebSocket 服务               通过 WebSocket 直连大屏
  运行本地 HTTP 服务（扫码分享）         扫大屏二维码获取 IP + Port + Token
  生成房间二维码                         或手动输入 IP:Port + Token
  持有完整白板状态（权威源）              加入后全量同步白板内容
                                        断线后本地继续书写，重连后追加同步
```

**协同特性**：
- 无需账号、无需云端，局域网匿名加入
- 大屏为权威状态源（Host），PAD 加入时全量拉取当前状态
- 协同同步内容：所有笔画（含图形）、橡皮擦操作、页面切换、新增 / 删除页
- 协同不同步：各设备的工具选择状态（画笔/橡皮/颜色等工具状态本地独立）
- 建议人数：6 台以内（无强制上限，受局域网性能影响）

## 2. 大屏 Host：开启协同流程

### 2.1 触发入口

工具栏 E 区"协同"按钮（🔗 图标），点击弹出协同 Bottom Sheet。

### 2.2 开启协同的完整流程

```
用户点击工具栏协同按钮（未开启协同时）
  │
  ▼
弹出协同 Bottom Sheet（Host 未开启视图）
  │
  │  ┌────────────────────────────────────────────────────────────┐
  │  │                    协同书写                                  │
  │  │                                                            │
  │  │  让同一 WiFi 下的平板扫码加入，实时共同书写。                   │
  │  │                                                            │
  │  │         [  开启协同  ]（color.brand.primary 背景按钮）       │
  │  │                                                            │
  │  └────────────────────────────────────────────────────────────┘
  │
  └─ 点击【开启协同】
       │
       ▼
       系统动作（约 500ms~1s）：
         - App 获取本机局域网 IP 地址
         - 启动本地 WebSocket 服务（默认端口 8765，随机 Token）
         - 生成房间二维码（编码内容：ws://[IP]:8765?token=[TOKEN]）
       │
       ▼
       Bottom Sheet 内容切换至【Host 已开启视图】（见 2.3 节）
```

### 2.4 二维码展示规范

| 属性 | 值 |
|------|---|
| 二维码尺寸 | `240dp × 240dp`（大屏底部 Sheet 内），见 qr-share-ux.md 的全屏展示模式 |
| 背景 | 白色（`#FFFFFF`），二维码本体深色（`#212121`）；注：二维码须使用白底深码，保证扫码识别 |
| 边框 | `4dp` 白色填充（quiet zone），`radius.sm`（8dp）圆角 |
| 刷新 | 二维码不自动过期，协同 Token 在会话结束前持续有效 |
| 文字标注 | 二维码下方：`ws://192.168.1.100:8765`（细节文字，`typography.size.caption`） |

│                                                                │
│  ─────────────  或手动输入  ─────────────                       │
│                                                                │
│  主机地址  [ 192.168.1.100:8765      ]                          │

│                                                                │
│  [  加入  ]                                                    │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 3.3 扫码加入流程

```
PAD 用户点击【扫描二维码加入】
  │
  ▼
调用系统相机扫码界面（或 App 内 ZXing 扫码）
  │
  ▼
扫描大屏上显示的协同二维码
  │
  ├─ 扫码失败（无法解码）→ 提示"无法识别二维码，请靠近后重试"，返回扫码界面
  │
  └─ 扫码成功，解码出 ws://[IP]:[Port]?token=[Token]
       │
       ▼
       自动填入主机地址和 Token 字段（无需用户手动输入）
       尝试建立 WebSocket 连接
       │
       ├─ 连接超时（3s 内未收到响应）→ 提示"无法连接，请确认与大屏在同一 WiFi 下"
       │       └─ 显示重试按钮
       │
       ├─ Token 验证失败 → 提示"加入失败，二维码已失效，请重新扫码"
       │
       └─ 连接成功
            │
            ▼
            全量同步当前白板内容（大屏推送当前 PageDataList 快照）
            同步进度 Loading（若数据量大，显示进度条）
            同步完成 → Bottom Sheet 切换至【PAD 参与者视图】（见 3.4）
```

## 4. 协同状态指示

### 4.1 工具栏协同按钮徽章

（详见 toolbar-component.md 7.2 节）

| 状态 | 徽章 |
|------|------|
| 未开启协同 | 无徽章 |
| 协同中（仅 Host 自己） | 绿色实心圆，`10dp` |
| 协同中（有其他设备在线） | 绿色圆 + 白色数字（在线总人数含本机） |
| 断线重连中 | 橙色圆 + 脉冲动画 |

### 4.2 协同状态 Banner（持久状态提示）

协同会话活跃期间，画布顶部显示一条细 Banner（不侵占画布主体区域）：

```
位置：画布区域顶部，紧贴屏幕顶边
高度：28dp（大屏）/ 24dp（PAD）
宽度：全屏宽度

正常连接态 Banner：
  背景：color.semantic.success.container（#1B5E20，深色模式）
  文字：" 🟢  协同中 · 3 台设备在线"
  字号：typography.size.caption（14sp 大屏）
  透明度：80%（不完全遮挡，保持视觉轻量）

断线重连态 Banner：
  背景：color.semantic.warning.container（#E65100，深色模式）
  文字：" 🟠  正在重新连接..."
  脉冲动画：Banner 背景色透明度在 60%~100% 之间循环（1s 周期）
```

> 画布顶部 35% 区域不可放置交互控件，但 Banner 仅为**纯展示信息**（无按钮），不需要用户点击，因此允许放置于顶部区域。Banner 不含任何可交互元素。

### 4.3 协同人数统计规则

- "在线设备数"包含 Host 本机
- PAD 断线（超过 5s 无心跳）后从在线列表移除，按钮徽章数字减少
- PAD 重连成功后重新计入，徽章数字增加

## 5. 主持人锁定功能

### 5.1 功能说明

Host 可开启"主持人控制"模式：
- **开启后**：只有 Host 可以翻页；参与者视角强制跟随 Host 当前页，无法自行翻页
- **关闭后**：所有设备可以独立浏览页面（书写仍然实时同步）

### 5.2 Host 端开关

开关位于协同 Bottom Sheet Host 视图底部（见 2.3 节）：

```
主持人控制  [ ○────  关闭 ]   （默认关闭）

开启状态：
主持人控制  [ ────●  开启 ]

开关样式：Material 3 Switch 组件
  关闭态：灰色轨道，白色滑块
  开启态：color.brand.primary 轨道，白色滑块
  切换动画：150ms standard 缓动
```

**开关触发后的系统行为**：
- 开启：广播 `LOCK_PAGES` 消息给所有 PAD，PAD 端立即锁定翻页功能
- 关闭：广播 `UNLOCK_PAGES` 消息，PAD 端恢复翻页功能

### 5.3 PAD 端锁定状态展示

当 Host 开启主持人控制后，PAD 端有以下视觉反馈：

**工具栏翻页区**：
```
翻页按钮（◀▶）视觉状态：禁用态图标（color.icon.disabled）
页码区域：正常显示，但点击无法打开页面管理 Bottom Sheet
在工具栏页码区右侧显示小锁图标：🔒（16dp，color.semantic.warning 颜色）
```

**协同 Bottom Sheet（PAD 参与者视图）**：
```
显示提示区（黄色 Banner）：
  ⚠️  主持人控制模式已开启
     您当前只能查看，无法独立翻页。
     视角将跟随主持人。
```

**主持人翻页时**：
- PAD 收到 `PAGE_CHANGE` 同步消息后，自动执行翻页动画（方向与 Host 一致）
- 翻页期间 PAD 的本地翻页按钮继续禁用

### 5.4 主持人控制下的书写权限

- 主持人控制模式**仅限制翻页**，不限制书写
- PAD 参与者在主持人控制模式下仍可在当前同步显示的页面上自由书写
- 若 Host 需要禁止参与者书写（只读），需在 v1.x 评估，v1.0 不实现

## 6. 断线重连逻辑与提示

### 6.1 断线检测

| 端 | 检测方式 | 判定条件 |
|----|---------|---------|
| Host 检测 PAD 断线 | 心跳机制（每 2s 发送 PING，超过 5s 未收到 PONG） | 认定该 PAD 断线，从在线列表移除 |
| PAD 检测自身断线 | WebSocket `onClose` / `onError` 回调 | 立即触发重连流程 |

### 6.2 PAD 断线后的行为

```
PAD WebSocket 连接断开
  │
  ▼
  ① 本地可继续书写（不阻塞用户操作）
     断线期间的操作存入本地"离线操作队列"
  │
  ▼
  ② 显示断线提示（非阻塞，不弹全屏弹窗）：
     工具栏协同按钮徽章切换为橙色（脉冲动画）
     画布顶部 Banner 切换为橙色：
       "🟠  协同已断线 · 正在重新连接...（第 1 次）"
  │
  ▼
  ③ 自动重连（指数退避）：
     第 1 次：1s 后重试
     第 2 次：2s 后重试
     第 3 次：4s 后重试
     第 4 次及以后：10s 间隔重试
     超过 5 次仍失败：Banner 文字更新为"🔴  协同连接失败，点击手动重连"
       （Banner 变为可点击区域，点击触发立即重连）
```

### 6.3 重连成功后的处理

```
WebSocket 重连成功
  │
  ▼
  ① 请求全量状态同步（拉取 Host 当前 PageDataList 最新状态）
  │
  ▼
  ② 合并离线期间的本地操作：
     将本地"离线操作队列"中的操作追加发送给 Host
     Host 以"后进操作追加"方式合并（简单追加，不做 OT/CRDT）
  │
  ▼
  ③ 清空离线操作队列
  │
  ▼
  ④ UI 反馈：
     画布顶部 Banner 切换回绿色（正常连接态）
     协同按钮徽章切换回绿色
     显示短暂 Toast："已重新连接"（2s 后消失）
```

### 6.4 Host 关闭 App / 结束协同

```
Host 点击协同 Bottom Sheet 中的【结束协同】按钮
  │
  ▼
  弹出确认 Bottom Sheet：
  "结束协同后，所有参与者将断开连接。"
  [ 取消 ]  [ 结束协同（color.semantic.error） ]
  │
  └─ 确认结束 →
       Host 广播 SESSION_END 消息给所有 PAD
       关闭 WebSocket 服务
       工具栏协同按钮徽章清除
       画布顶部 Banner 消失

PAD 收到 SESSION_END 消息 / Host 直接关闭 App：
  显示 Toast："大屏已结束协同"（3s）
  协同按钮徽章清除（切换回无徽章状态）
  顶部 Banner 消失
  PAD 可继续在本地书写（已有内容保留）
```

## 7. 多用户光标与标注显示

### 7.1 设备颜色分配

房间中每台设备分配唯一标识颜色（由 Host 分配，加入时通过 JOIN_ACK 消息返回）：

| 设备序号 | 标识颜色 | 色值（深色主题） |
|---------|---------|--------------|
| Host（大屏） | 蓝色 | `#64B5F6` |
| 参与者 1 | 绿色 | `#81C784` |
| 参与者 2 | 橙色 | `#FFB74D` |
| 参与者 3 | 紫色 | `#CE93D8` |
| 参与者 4 | 黄色 | `#FFF176` |
| 参与者 5 | 粉色 | `#F48FB1` |

超过 6 台设备时颜色循环复用（v1.0 建议 6 台以内，不做特殊处理）。

### 7.2 远端光标显示规范

每台远端设备在当前书写位置显示一个彩圈光标：

```
彩圈光标样式：
  形状：圆形
  直径：16dp（大屏）/ 12dp（PAD）
  外圈：设备标识颜色，宽度 2dp
  内圈：透明（中空）
  中心：设备标识颜色实心小点，直径 4dp
  透明度：70%（半透明，不遮挡书写内容）

光标位置：
  跟随远端设备当前触点坐标（归一化坐标还原后对应本机像素坐标）
  ACTION_UP 时（手指抬起）：光标保留在最后书写位置，3s 后淡出消失

光标标签：
  光标上方显示设备简称（如 "PAD-1"、"大屏"）
  字号：typography.size.caption（12sp）
  颜色：设备标识颜色
  仅在该设备正在书写时显示，3s 无操作后标签淡出
```

### 7.3 协同笔画颜色

- 远端设备的笔画颜色：使用**对方设备自己选择的书写颜色**（不强制使用设备标识颜色）
- 设备标识颜色仅用于光标彩圈，不影响笔画颜色
- 设计目的：保证书写颜色忠实反映参与者的意图（用对方选的颜色），用彩圈辅助区分归属

## 8. 协同 Bottom Sheet 完整规范

### 8.1 Bottom Sheet 通用属性

| 属性 | 值 |
|------|---|
| 触发方式 | 工具栏协同按钮（🔗）点击 |
| 面板形式 | Bottom Sheet，带遮罩 |
| 顶部圆角 | `radius.xl`（24dp，协同面板视觉上更独立） |
| 遮罩 | `color.overlay.scrim`（50% 黑色） |
| 默认高度 | 屏幕高度的 `45%` |
| 可拖拽高度范围 | `35%` ~ `60%` |
| 关闭方式 | 向下拖拽 / 点击遮罩 / 内部显式关闭按钮 |

### 8.2 状态流转

```
                   ┌─────────────────────────────┐
  [Host 未开启] ──→ │ Host 已开启                   │ ←── 开启协同按钮
                   │  · 房间二维码                  │
  [结束协同]   ←── │  · 在线设备列表                │
                   │  · 主持人控制开关               │
                   └─────────────────────────────┘

                   ┌─────────────────────────────┐
  [PAD 未加入] ──→ │ PAD 已加入                   │ ←── 扫码加入
                   │  · 房间信息                   │
  [离开房间]  ←── │  · 在线列表                   │
                   │  · 锁定状态提示               │
                   └─────────────────────────────┘
```

### 8.3 在线设备列表规范

```
每个设备条目：
  高度：48dp
  布局：[彩圈 16dp] [间距 12dp] [设备名称] [设备角色标签（主持人/参与者）]

  彩圈：设备标识颜色实心圆，直径 16dp
  设备名称：typography.size.body（18sp 大屏），typography.weight.medium
  角色标签：小标签，typography.size.caption，
