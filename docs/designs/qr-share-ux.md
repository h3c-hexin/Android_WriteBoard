## 1. 功能概述

```
扫码带走（QR Share）流程：

  大屏（App）                             手机（无需安装 App）
  ─────────────────────                  ───────────────────────
  用户点击"分享"按钮                      手机扫码
      ↓                                       ↓
  App 渲染当前所有页面为 PNG 图片          系统浏览器打开 URL
  启动本地 HTTP 服务                      浏览器显示所有白板页截图
  生成含一次性 Token 的 URL              支持长按保存单张图片
  显示 URL 对应的二维码                  支持一键下载 PDF（含所有页）
  二维码在 Bottom Sheet 或全屏展示
```

**关键约束**：
- 手机须在**同一 WiFi 下**，否则无法访问大屏本地 HTTP 服务
- 手机无需安装任何 App，纯浏览器体验
- URL 包含一次性 Token，同一 WiFi 下其他设备扫码需重新点击分享生成新 Token

## 2. 分享按钮点击后的 Bottom Sheet 流程

### 2.1 触发入口

工具栏 E 区"分享"按钮（📤 图标），点击后弹出分享 Bottom Sheet。

### 2.2 分享准备阶段

```
用户点击工具栏分享按钮
  │
  ▼
  ① 检查 HTTP 服务是否已启动
     - 若未启动：启动本地 HTTP 服务（约 200ms~500ms）
     - 若已启动（如之前开启过分享或协同）：复用

  ② 异步渲染当前所有页面为 PNG 图片（后台任务）
     - 每页独立渲染（Canvas 截图，包含背景 + 笔画）
     - 渲染在后台线程，主线程不阻塞
     - 若页数 ≤ 5：通常 < 500ms 完成（大屏分辨率下）
     - 若页数较多：先显示 Bottom Sheet，后台渲染，进度更新

  ③ 生成一次性 Token（见第 6 节）

  ④ 构建分享 URL：
     http://[本机局域网 IP]:8080/board?token=[ONE_TIME_TOKEN]
  │
  ▼
  弹出分享 Bottom Sheet（见 2.3 节）
```

### 2.3 分享 Bottom Sheet 布局

```
┌──────────────────────────────────────────────────────────────────────┐
│  ╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌（拖拽条）                                     │
│                                                                      │
│  分享白板                                                             │
│                                                                      │
│  ┌──────────────────────────────────────────────────────────────┐    │
│  │                                                              │    │
│  │          二维码（200dp × 200dp）                              │    │
│  │                                                              │    │
│  └──────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  用手机扫描此二维码查看白板内容                                          │
│  http://192.168.1.100:8080/board?token=Ax3kZ                        │
│  ⚠️  手机须连接同一 WiFi                                              │
│                                                                      │
│  导出格式                                                             │
│  [ PNG · 当前页 ]   [ PNG · 全部页 ]   [ PDF · 全部页 ]               │
```

### 2.4 Bottom Sheet 规范

| 属性 | 值 |
|------|---|
| 顶部圆角 | `radius.lg`（16dp） |
| 遮罩 | `color.overlay.scrim` |
| 默认高度 | 屏幕高度的 `50%` |
| 关闭方式 | 向下拖拽 / 点击遮罩 |
| 关闭时 HTTP 服务 | 不关闭（用户关闭 Bottom Sheet 不等于结束分享，见第 7 节） |

## 3. 导出格式选择

### 3.1 三种导出格式

| 格式 | 说明 | 手机端体验 |
|------|------|---------|
| PNG · 当前页 | 仅导出当前显示的页面为单张 PNG | 浏览器显示单张图片，可长按保存 |
| PNG · 全部页（默认） | 所有页面各自导出为 PNG | 浏览器显示所有图片列表，可逐张长按保存 |
| PDF · 全部页 | 所有页面合并为单个 PDF 文件 | 浏览器提供"下载 PDF"按钮，点击下载 |

### 3.2 格式选择 UI

选项卡式横排按钮（3 个）：

```
[ PNG · 当前页 ]   [ PNG · 全部页 ]   [ PDF · 全部页 ]

  选中态：color.brand.primary 背景，白色文字，radius.sm 圆角
  未选中：透明背景，color.icon.default 文字，border 1dp color.divider
  字号：typography.size.label（16sp 大屏）
  高度：40dp（大屏）/ 36dp（PAD）
```

格式切换后：
- 二维码 URL 中追加格式参数（`&format=png_all` / `&format=png_current` / `&format=pdf`）
- 手机端访问该 URL 时，服务端根据格式参数返回对应内容
- 二维码内容更新，但**二维码外观视觉立即刷新**（原 Token 失效，新 Token 生成）

### 3.3 本地导出（通过系统分享）

"更多"菜单中"导出"功能通过系统分享发送（不依赖 HTTP 服务），详见 file-management-ux.md。此处分享按钮的"导出"为**扫码方式分享**，非系统文件导出。

## 4. 大屏二维码展示规范

### 4.1 Bottom Sheet 内二维码（常规视图）

| 属性 | 值 |
|------|---|
| 二维码尺寸 | `200dp × 200dp` |
| 背景 | 白色（`#FFFFFF`），保证扫码识别率 |
| 安全边距（quiet zone） | 4 个模块宽（自动包含在二维码内），视觉上约 `8dp` 白色外框 |
| 圆角 | `radius.sm`（8dp） |
| 容错级别 | Level Q（25% 容错），兼顾分辨率与识别距离 |
| 码色 | `#212121`（深色），白底深码 |

### 4.2 全屏二维码展示模式

用户点击"放大二维码（全屏展示）"后，进入全屏二维码模式（用于大屏站立使用时手机扫码更方便）：

```
全屏遮罩（不是 Bottom Sheet，而是 Activity 级覆盖层）：

  ┌───────────────────────────────────────────────────────────────────┐
  │  背景：rgba(0,0,0,0.92)，全屏覆盖，画布不可见                        │
  │                                                                   │
  │                    用手机扫码查看白板                                │
  │              （typography.size.display，白色）                     │
  │                                                                   │
  │         ┌──────────────────────────────────────────┐              │
  │         │                                          │              │
  │         │                                          │              │
  │         │          二维码（320dp × 320dp）            │              │
  │         │                                          │              │
  │         │                                          │              │
  │         └──────────────────────────────────────────┘              │
  │                                                                   │
  │          http://192.168.1.100:8080/board?token=Ax3kZ              │
  │              （typography.size.body，白色，居中）                   │
  │                                                                   │
  │          ⚠️  手机须连接同一 WiFi                                    │
  │              （typography.size.label，color.semantic.warning）     │
  │                                                                   │
  │                    [  关闭全屏二维码  ]                              │
  │              （底部文字按钮，white 颜色，typography.size.title）      │
  │              触控热区：64dp × 64dp，位于屏幕底部安全区内               │
  │                                                                   │
  └───────────────────────────────────────────────────────────────────┘
```

全屏二维码规格：

| 属性 | 值 |
|------|---|
| 二维码尺寸 | `320dp × 320dp`（大屏，1 米外仍清晰可扫） |
| 背景 | 白色圆角框（`radius.lg` 16dp，白色背景），二维码深色 |
| 最小扫码距离 | 按 QR Code 标准，`320dp × 320dp` 在 65" 大屏约为 28cm 实际尺寸，扫码距离 0.3~2m |
| 关闭方式 | 点击"关闭全屏二维码"按钮（位于屏幕底部 65% 安全区内） |

## 5. 手机扫码浏览体验

### 5.1 浏览器端 Web 页面设计原则

（此部分为前端 HTML 页面规范，由后端 HTTP 服务渲染，非 Android UI，但 UX 需统一规划）

```
页面结构（PNG 全部页模式）：

  顶部：
    App 名称 "PaintBoard" + 页面数量说明（如"共 5 页白板"）
    白板创建时间或当前时间

  内容区：
    各页白板图片纵向排列，响应式宽度（100vw，保持比例）
    图片间距：16px
    页码标签：图片下方居中（"第 1 页 / 共 5 页"）

  底部操作栏（固定）：
    [ 下载 PDF（全部页）] 按钮（仅 PDF 模式显示）
    [ 保存当前图片 ] 按钮（仅 PNG 当前页模式显示）
    提示文字："长按图片可单独保存"

  响应式适配：
    移动端竖屏：100% 宽度图片，单列纵向排列
    平板横屏：50% 宽度图片，双列网格
```

### 5.2 手机端操作提示

浏览器页面顶部提示横幅（首次访问时显示）：
```
"长按图片可保存到相册，或点击底部按钮下载 PDF"
点击横幅可关闭
```

### 5.3 访问失效处理

| 情况 | 浏览器显示 |
|------|---------|
| Token 已失效（一次性 Token 被使用过）| 提示"此分享链接已失效，请在大屏重新生成二维码" |
| HTTP 服务已关闭（App 退后台超 10 分钟） | 浏览器连接超时，原生错误页，无法优化 |
| 不在同一 WiFi | 浏览器连接超时，原生错误页 |

## 6. 一次性 Token 安全机制

### 6.1 Token 生成规则

```
Token 生成：
  算法：随机生成 8 字节（64 bit），Base62 编码
  输出：约 11 个字符（字母数字混合，如 "Ax3kZ9mQ2r"）
  唯一性：UUID 随机，碰撞概率极低（会议场景下可接受）
  有效期：至下次生成新 Token（点击分享按钮 / 切换导出格式）为止

Token 存储：
  内存中（不持久化），App 重启后失效
  协同 Token 与分享 Token 独立管理，不互用
```

### 6.2 一次性 Token 使用规则

**"一次性"的语义**：

| 版本规划 | 说明 |
|---------|------|
| v1.0 | Token 有效，直到用户**重新点击分享按钮**或**切换导出格式**（生成新 Token），旧 Token 失效；同一 URL 可被多次扫码访问（适合会议场景，与会者可多次下载）|
| v1.x 评估 | 严格一次性（扫码一次后立即失效）；适合更高安全要求场景 |

> v1.0 说明：Token 不是"扫一次即失效"，而是"新分享生成新 Token，旧 Token 自动失效"。这样同会议室的多名参与者可以依次扫码，无需每人重新生成。同一 WiFi 下不知道 URL 的人无法访问（URL 不广播，仅通过二维码传递）。

### 6.3 Token 展示规范

Token URL 在 Bottom Sheet 中以小字显示：

```
http://192.168.1.100:8080/board?token=Ax3kZ9mQ2r

字号：typography.size.caption（14sp 大屏）
颜色：color.icon.default（次要信息色）
可选功能：点击 URL 复制到剪贴板（Toast："已复制链接"）
```

## 7. HTTP 服务生命周期

### 7.1 服务启动时机

| 触发条件 | 行为 |
|---------|------|
| 用户首次点击分享按钮 | 启动本地 HTTP 服务（端口 8080，若被占用自动递增至 8081、8082...）|
| 协同和分享共用 HTTP 服务 | 复用同一服务实例，仅路由不同（`/board` vs `/collab`）|

### 7.2 服务关闭时机

| 触发条件 | 行为 |
|---------|------|
| App 退到后台 10 分钟后 | 服务关闭（Android WorkManager 延迟任务，App 在后台时防止资源占用） |
| App 被用户强制终止 | 服务随 App 进程结束 |
| 用户在"更多"菜单手动停止（v1.1 考虑） | 服务关闭，Token 失效 |

**注意**：当前 Bottom Sheet 关闭**不会关闭 HTTP 服务**，用户关闭分享 Bottom Sheet 后，之前生成的 URL 仍然有效。这允许用户在分享二维码被扫描后、回到书写界面继续使用。

### 7.3 服务状态可见性

- HTTP 服务运行状态对用户不可见（无持久状态指示器）
- 服务意外停止时（被系统回收），下次点击分享按钮会自动重启

## 8. 错误与异常处理

| 错误场景 | 提示方式 | 提示文案 | 恢复操作 |
|---------|---------|---------|---------|
| 无法获取局域网 IP（未连接 WiFi） | Toast（非阻塞） | "未连接 WiFi，无法开启分享" | — |
| HTTP 服务启动失败（端口冲突严重） | Bottom Sheet 内 Error Banner | "服务启动失败，请稍后重试" | "重试"按钮 |
| 页面渲染失败（内存不足） | Bottom Sheet 内 Error Banner | "页面截图失败，请关闭其他应用后重试" | "重试"按钮 |
| PDF 生成失败 | Toast | "PDF 生成失败" | 切换为 PNG 格式 |

错误提示详细规范见 error-states-ux.md。
