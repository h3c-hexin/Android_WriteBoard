# PaintBoard — 产品需求文档 (PRD)

> 版本：v0.2
> 日期：2026-03-23
> 状态：草稿，待确认
> 修订说明：基于竞品分析和产品方向讨论，从"通用绘画 App"调整为"商务会议白板"

---

## 1. 产品概述

**PaintBoard** 是一款面向 Android 平台的**商务会议白板应用**，主要运行在会议室商用大屏（55"~86"）上，同时支持安装在 PAD 上，两端功能完全一致。

核心场景：多人围绕大屏开会，用手指/触控笔实时书写讨论；会后通过扫码将白板内容带走；同一 WiFi 下的大屏与 PAD 可实时协同书写。

产品定位：**功能刚好够用的会议白板**，不做复杂绘画工具，专注会议书写流畅度和协同分享体验。

---

## 2. 用户角色

| 角色 | 设备 | 描述 |
|------|------|------|
| 会议主持 | 大屏 / PAD | 发起会议白板，管理页面，控制协同会话 |
| 会议参与者 | PAD | 扫码加入大屏协同会话，在自己设备上实时共同书写 |
| 内容查看者 | 手机（任意设备）| 扫码后用浏览器查看/下载白板内容图片，无需安装 App |

## 3. 功能需求

### 3.1 书写工具

#### F01 — 自由书写（笔刷）
- **描述**：手指或触控笔在画布上自由书写，会议白板场景优先保证流畅度
- **笔刷参数**：
  - 粗细：细 / 中 / 粗 三档快速切换（也可滑块微调）
  - 颜色：见 F05
- **平滑处理**：贝塞尔曲线插值，手指书写时做适度平滑
- **压感支持**：设备支持时自动启用，不支持则固定粗细

#### F02 — 橡皮擦
- **描述**：擦除画布上的内容
- **参数**：擦除半径三档（小 / 中 / 大）
- **自然书写/擦除识别**（按设备类型区分）：
  - **大屏模式**：基于触点面积实时判断——小面积触点（指尖）视为书写，大面积触点（手掌）视为擦除；各触点簇独立判断，多人同时操作互不干扰；两人手掌过近导致触点簇合并的极端场景暂不处理
  - **PAD 模式**：基于输入设备类型判断——触控笔触点视为书写，手指触点视为擦除；符合"拿笔写、手指擦"的真实书写直觉，无需切换任何工具状态
- **清除当前页**：橡皮擦工具激活时，子面板提供"清除当前页全部内容"滑动控件
  - 交互方式：滑动解锁式控件（类似 iOS 关机滑块），手指须主动划过整个滑轨才触发，防止误触
  - 无弹窗确认，滑动即执行，轻量不打断节奏
  - **支持撤销**：清除操作纳入 Undo/Redo 栈，可通过撤销恢复全部内容

#### F03 — 图形工具
- **v1.0 支持图形**：
  - 直线
  - 箭头（从 v1.1 提前，会议场景高频）
  - 矩形（空心 / 填充）
  - 圆形/椭圆（空心 / 填充）
  - 三角形（空心 / 填充）
  - 菱形（空心 / 填充，流程图判断节点常用）
- **v1.1 扩展**：双向箭头
- **操作方式**：拖拽确定起点和终点/大小，松手完成
- **图形自动矫正**：手绘时识别并自动矫正为标准形状；**可通过设置开关关闭**

#### ~~F04 — 文字工具~~
> **已移除**：大屏场景下弹出软键盘体验差，且会议场景以手写为主，不做文字输入工具。手写转文字（F12 手写识别）可作为替代方案在 v1.5 考虑。

#### F05 — 颜色选择
- **工具栏常驻**：4 个快捷色块（黑、红、蓝、当前自定义色），单击即换，无需弹窗
- **二级色板**：点击"更多颜色"按钮 → 从底部弹出色板面板，包含：
  - 预设色：20 种常用高对比度颜色（5×4 宫格）
  - 最近使用：最近 5 种用过的颜色
  - 自定义：HSV 调色盘（供有需要的用户精确选色）
- **选色后**：所选颜色替换工具栏第 4 个"自定义色"槽位，面板自动收起
- **笔刷粗细**：与颜色并排，三档图标切换

### 3.2 撤销/重做

#### F06 — Undo / Redo
- **步数**：最多 50 步
- **范围**：所有书写、图形、文字操作
- **触发方式**：工具栏按钮（不支持手势触发，避免与书写手势冲突）

### 3.3 多页白板管理

#### F07 — 页面管理
- **描述**：一次会议可以创建多张白板页，类似幻灯片
- **操作**：
  - 新建页（当前页末尾添加）
  - 删除页（确认后）
  - 页面缩略图列表，点击切换页面
  - 每个缩略图旁提供上移/下移箭头按钮调整顺序（不使用拖拽手势）
- **上限**：最多 20 页
- **页面背景**：纯白（默认）/ 网格 / 横线（三种选项，按页设置）

### 3.4 局域网协同书写

#### F08 — LAN 实时协同
- **描述**：同一 WiFi 下多台设备实时共同书写，无需云端账号
- **架构**：大屏作为 Host，运行本地 WebSocket 服务；PAD 通过扫码获取大屏 IP，直连加入
- **发起协同**（Host 端）：
  - 点击"开启协同"→ 生成房间二维码（包含局域网 IP + 端口 + 房间 Token）
  - 显示当前在线设备数
- **加入协同**（PAD 端）：
  - 扫描大屏上的二维码
  - 或手动输入 IP:Port + Token
  - 加入后当前画板内容自动同步
- **同步内容**：所有笔画、图形操作实时同步；页面切换同步
- **主持人锁定**：Host 可开启"主持人控制"模式，开启后只有 Host 可以翻页，参与者视角跟随 Host；关闭后参与者可自由浏览页面
- **用户标识**：不同设备的笔画颜色光标以小彩圈区分
- **断线处理**：PAD 断线后本地可继续书写，重连后自动合并（后进操作追加）
- **人数**：建议 6 台以内（无强制限制，视局域网性能）

### 3.5 扫码带走

#### F09 — 内容分享（扫码带走）
- **描述**：会议结束后生成二维码，手机扫码后用浏览器打开即可查看/保存白板内容
- **架构**：大屏运行轻量 HTTP 服务，二维码编码局域网 URL
- **手机端体验**：
  - 扫码 → 系统浏览器打开
  - 显示所有页面的白板截图（响应式排版）
  - 支持长按保存单张图片
  - 支持一键下载 PDF（含所有页）
- **注意**：手机需在同一 WiFi 下；手机无需安装任何 App
- **访问安全**：URL 中包含一次性 Token，防止同 WiFi 下其他人误访问
- **服务生命周期**：App 在前台时 HTTP 服务持续运行；App 退到后台 10 分钟后服务关闭

### 3.6 内容保存

#### F10 — 文件保存与打开

- **保存**：用户主动将当前白板（所有页面 + 笔画数据）保存为 `.pb` 文件
  - 可自定义文件名（默认按日期时间命名，如 `会议_2026-03-23_14:32`）
  - 保存位置：设备本地存储，用户可自选目录
  - `.pb` 文件可通过 U 盘、局域网、蓝牙等方式传给其他设备
- **打开**：从设备存储选择 `.pb` 文件打开
  - 若当前有未保存内容，弹出提示"是否先保存当前内容？"[保存][不保存][取消]
  - 打开后完整还原所有页面和笔画
  - 任何装有本 App 的设备均可打开同一 `.pb` 文件
- **新建白板**：清除所有页面，重置到初始化状态（1 张空白页，默认背景）
  - 执行前弹出提示"是否保存当前内容？"[保存][不保存][取消]
  - 确认后清除全部内容，**不可撤销**（区别于清除当前页）
- **崩溃恢复**：App 每 2 分钟静默写入一个临时恢复文件（用户不可见）
  - App 重启时若检测到临时文件，弹出一次性提示"发现上次未保存的内容，是否恢复？"
  - 用户确认恢复 / 拒绝则删除临时文件，不再提示
- **导出**：
  - 导出当前页为图片（PNG）
  - 导出全部页为 PDF
  - 通过系统分享发送（不影响 `.pb` 文件）

#### F11 — 导入图片作为底图（v1.1）
- 从相册/文件管理器选择图片插入当前页作为背景
- 背景图不可书写，只作参考底图

#### F12 — 手写便利贴（v1.5）
- **形态**：有背景色的可移动卡片容器，内部通过手写笔画填写内容
- **创建**：选择便利贴工具 → 点击画布放置卡片 → 自动进入卡片书写模式 → 点击卡片外部完成
- **卡片操作按钮**：完成书写后，卡片右上角常驻三个图标按钮：✏️编辑（重新进入书写模式）、✋移动（显示拖拽手柄，拖动整张卡片）、🗑删除（确认后删除）；无任何手势触发
- **卡片颜色**：6 种背景色（黄/蓝/绿/粉/橙/白），创建时可选，创建后可通过编辑按钮更改
- **卡片尺寸**：固定尺寸（约 200×200dp），保持统一整洁
- **协同同步**：便利贴作为对象参与实时同步，包括位置、内容、颜色变更

**架构预留要求（v1.0 开发阶段必须考虑）**：
> 便利贴引入"**可移动对象层**"概念：区别于主画布的像素笔画，便利贴是带有位置、尺寸、颜色属性的独立对象，其内部笔画坐标相对于卡片局部坐标系，移动卡片时整体平移。
> v1.0 架构设计时须预留：
> - 画布数据模型支持"对象列表"与"笔画列表"并存
> - 渲染层支持对象与笔画的分层绘制（对象层叠加在笔画层之上）
> - 协同同步协议支持对象级别的增删改操作（不只是笔画 append）

## 4. 非功能需求

| 需求 | 指标 |
|------|------|
| 书写延迟 | 触摸到屏幕显示 < 16ms（60fps） |
| 协同同步延迟 | 局域网下单笔画同步 < 100ms |
| 启动时间 | 冷启动 < 2s |
| 最低 Android 版本 | Android 8.0 (API 26) |
| 目标 Android 版本 | Android 14 (API 34) |
| 屏幕适配 | 横屏为主（大屏/PAD），工具栏适配 1 米外站立操作（图标 ≥ 48dp，间距充足）|
| 多点触控 | 最少支持 10 点同时触控（大屏硬件通常支持 20 点）|
| 设备模式自动识别 | App 启动时根据屏幕对角线尺寸自动切换模式：> 30" 为大屏模式，≤ 30" 为 PAD 模式；设置中提供手动覆盖选项 |
| 输入设备识别 | PAD 模式：区分触控笔与手指（MotionEvent.getToolType()）；大屏模式：识别触点接触面积（MotionEvent.getTouchMajor/Minor()）|
| 深色模式 | 支持（会议室可能关灯使用）|
| 离线可用 | 核心书写功能完全离线可用，协同和扫码带走需要局域网 |

## 6. 用户流程

### 场景 A：会议主持开启白板
```
启动 App
  └── 直接进入白板（自动恢复上次会话内容）

白板主界面
  ├── 书写/画图
  ├── 切换页面 / 新增页面
  ├── 点击"协同" → 显示协同二维码，PAD 扫码加入
  ├── 点击"分享" → 显示带走二维码，手机扫码获取图片
  ├── 点击"新建" → 清空开始新白板（可选保存当前）
  └── 自动保存
```

### 场景 B：PAD 加入协同书写
```
PAD 启动 App
  └── 首页点击"加入协同"
        └── 扫描大屏上的协同二维码
              └── 加入会话 → 实时同步白板内容 → 共同书写
```

### 场景 C：手机扫码带走内容
```
手机（无需安装 App）
  └── 扫描大屏上的"分享"二维码
        └── 系统浏览器打开
              ├── 查看所有白板页截图
              ├── 长按保存单张图片
              └── 点击"下载 PDF"
```

## 7. 已确认决策

> 关于画布缩放/移动：所有版本均不支持写作时缩放或平移画布。大屏是多人共用视图，缩放会打乱所有人视角，且与撤销/重做手势冲突。空间不足时通过多页管理解决。PAD 协同时视图与大屏同步，不支持独立缩放。回顾模式（查看时缩放）暂不做，待后续版本再评估。

| 议题 | 决策 |
|------|------|
| 协同账号体系 | 无需账号，局域网内匿名加入 |
| 协同网络方案 | 局域网（同一 WiFi），大屏 Host 本地 WebSocket |
| 扫码带走内容 | 静态图片/PDF，手机用浏览器打开，无需装 App |
| 扫码带走安全 | URL 含一次性 Token，防止他人误访问 |
| 图层系统 | 不做，会议场景不适用 |
| 颜色系统 | 工具栏 4 个快捷色（黑/红/蓝/自定义槽）+ 二级色板（20 预设色 + HSV 自定义）|
| 文字工具 | 不做，大屏弹键盘体验差，手写为主 |
| 主要运行设备 | 商用 Android 大屏（55"~86"）+ PAD，功能一致 |
| 启动行为 | 无首页，直接进入画板，自动恢复上次会话 |
| 新建白板 | 清除所有页面回到初始状态（1张空白页），操作前提示保存，不可撤销 |
| 历史记录 | 不做自动历史，改为用户主动保存/打开 `.pb` 文件，文件可跨设备传递 |
| 清屏功能 | 不作为独立功能，通过橡皮擦子面板的滑动控件"清除当前页"实现，支持撤销 |
| 主持人锁定 | 支持，Host 可开启控制模式限制参与者翻页；页面管理（新建/删除/排序）暂定仅 Host 可操作，细节在 UX 阶段定义 |
| 图形自动矫正 | 支持，但提供设置开关可关闭 |
| 网格线间距 | UI 阶段确认 |
